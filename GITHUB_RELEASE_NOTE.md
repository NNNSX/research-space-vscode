这是一次面向文件拆解、AI 消费链路和误删防护的 alpha 汇总更新。

## 新增

- 新增“文件爆炸”AI 工具主线，可从工具节点触发文档拆解并在画布上生成结构化节点组
- 文件爆炸支持从 PDF 扩展到 DOCX、PPT/PPTX、图片、XLS/XLSX 等多种输入
- 新增 MinerU 在线解析配置入口，支持 Token、模式、模型版本、轮询参数与本地 fallback 设置
- 拆解结果会生成文档关系索引，帮助后续 AI 按“关系索引 → 文本 → 图片”的顺序读取内容
- PPT/PPTX 增加整页预览渲染链路，优先保留幻灯片级视觉结果

## 修复与优化

- 多次拆解同一文件时默认保留旧结果，并将新结果自动排到旧结果下方
- PPT/PPTX 整页渲染改为更稳定的 PDF 到逐页 PNG 链路，并补齐 PowerPoint / LibreOffice / Quick Look 等后端检测与错误提示
- MinerU 在线轮询、下载、鉴权、限流、超时与结果缺失等错误提示更明确
- 文件爆炸节点的状态、配置提示、运行前检查和执行阶段提示更紧凑清晰
- 画布节点、连线、节点组、画板、暂存项、任务项、设置项和对话清空等删除入口统一补齐确认层
- 顶栏排版收口，窄宽度下保存状态、标题和按钮区不再挤成多行

## 注意事项

- 这是 alpha 测试版，建议先备份工作区，再在副本中试用
- PPT/PPTX 整页预览依赖本机可用的 PowerPoint、LibreOffice 或系统预览能力；缺失时会给出提示或回退到可用结果
- MinerU Token 只应配置在本机 VS Code 设置中，不应写入源码、工作区文件或发布说明
