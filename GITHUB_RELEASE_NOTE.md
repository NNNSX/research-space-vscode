这是一次面向 AI 工具、多模态生成、来源引用、文件转换与画布稳定性的 alpha 汇总更新。

## 新增

- 文本类 AI 输出改为正文内引用，尽量在相关句子或段落末尾嵌入 `[资料1]`、`[资料2]` 等来源标记。
- 选中节点后新增“导出 MD”，可把当前选区按画布位置合并导出为 Markdown。
- 选中论文、笔记、AI 输出、图片、音频等节点后，会显示轻量下一步建议入口。
- 多模态图像工具新增 GPT Image 2，并继续支持 Gemini、Doubao 等图像模型。
- TTS、STT、会议转写、图像和视频工具的调用提示、错误提示与输出语义进一步收口。
- 文件转换支持 PDF、Word、PPT、图片、XLS / XLSX 等输入，表格可转 Markdown 或 TeX。
- 新增转换环境诊断与画布数据健康检查，降低环境问题和结构问题排查成本。

## 修复与优化

- AI 文本输出不再只在末尾堆叠来源说明，来源追踪信息仍保留在节点元数据中，便于导出和追溯。
- 自定义文本工具和 AI 编排提示词会继续兼容正文内来源引用。
- 图像请求、音频请求、视频请求拆分为更清晰的执行模块，错误信息更易读。
- 修正 GPT Image 2 生成与编辑请求链路，减少长耗时生成时的网络失败误判。
- 高成本图像 / 视频工具增加确认提示，重复运行会保留新结果，避免覆盖用户产出。
- 优化多节点画布缩放与拖拽流畅度，远视角下自动进入摘要态。
- 画板在摘要态显示置顶大标题，低缩放下可调边界角点更容易捕捉。
- 节点、连线、画板、节点组、暂存项、任务项、配置项和对话清空等删除入口补齐确认层。

## 注意事项

- 建议在试用 alpha 版本前备份工作区。
- PPT / Word 转 PNG 依赖本机可用的 PowerPoint、Microsoft Word 或 LibreOffice / soffice；缺失时可在设置中运行转换环境诊断。
- MinerU Token 和模型 API Key 只应配置在本机 VS Code 设置中，不应写入项目文件。
- 中风险画布修复目前只展示计划，不会自动执行。
