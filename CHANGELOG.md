# Changelog

All notable changes to **Research Space** are documented here.

## [2.1.1-alpha.1] — 2026-04-21

- **发布汇总版本：收口自 `v2.1.0-alpha.41` 以来的蓝图主线改造** — 这不是单个小补丁版，而是把 `alpha.42` 到 `alpha.140` 期间已落地的有效改动统一汇总进新的发布节点：重点包括旧蓝图实例/旧画布严格迁移、蓝图 overlay 外壳与整实例拖拽、输出槽位历史与失败后继续、蓝图最终输出持久化/恢复链路、隐式 output slot 自动补齐、输出节点外置与重开不再掉线、输入/输出占位样式与自动测高收口，以及 Copilot 默认模型改为 `gpt-4.1`、创建蓝图弹窗文本选择误关修复、普通文件节点 footer 回归修复等。详细对比说明请见 GitHub Release Notes

## [2.1.0-alpha.140] — 2026-04-20

- **删除输入占位 footer 里重复的提示块** — 输入占位内部已经有“当前输入 / 接受类型 / 定位输入节点”等主信息，这轮把 footer 里重复出现的摘要提示、状态 chips 和定位按钮去掉，只保留更轻的槽位标题与当前输入状态，避免同一信息在节点里重复堆叠

## [2.1.0-alpha.139] — 2026-04-20

- **修复 alpha.138 占位节点底部边界无限下长的回归** — 上一轮为了把 footer 纳入 placeholder 自动测高，误把测量 ref 挂到了会跟节点当前高度一起伸展的容器上，结果测高链路把“当前高度”也当成了内容高度，触发持续增高。现在改回只测量占位卡片的自然内容骨架，并把 footer 一起纳入同一测量容器，既能覆盖多行底部提示，也不会再无限下长

## [2.1.0-alpha.138] — 2026-04-20

- **修复占位节点底部多行提示被遮盖** — 根因不在文案本身，而在 blueprint placeholder 的自动测高只测了 header + body，没有把 footer 算进去；当底部提示或状态说明变成两行以上时，节点高度不会跟着长，结果 footer 会被裁住。现在占位节点的自动测高已覆盖整张卡片内容，底部多行提示会随节点一起扩高，不再被遮盖

## [2.1.0-alpha.137] — 2026-04-20

- **输入占位继续按输出占位母版收口** — 这轮不再让输入/输出占位在 footer 里各走一套结构，而是直接把占位底部的槽位标题、当前状态、摘要说明、状态 chips 和定位动作统一为同一套 Placeholder Shell；输入占位仅替换为“接受类型 / 必填 / 多输入”等少量输入语义，避免继续出现“看起来像两种不同节点”的结构分叉

## [2.1.0-alpha.136] — 2026-04-20

- **输入/输出占位开始共用同一套 Placeholder Shell** — 这轮不再零碎补样式，而是把占位的标题区状态 badge、主体状态区、动作按钮入口统一到同一套骨架上；输入/输出现在共享相同的 shell，只保留“当前输入/当前输出”“定位输入节点/定位上游节点”等少量语义差异，从结构上收口“为什么总是看起来不像同一种占位节点”的问题

## [2.1.0-alpha.135] — 2026-04-20

- **修复蓝图创建弹窗里拖拽选中文字会导致弹窗误关闭的隐性 bug** — 根因不是蓝图草稿本身崩了，而是弹窗 backdrop 仍按“mouseup 后 click 落在遮罩上就关闭”处理；当你从输入框/说明文本里拖拽选中文字、鼠标释放到弹窗外时，就会被误判成点了遮罩。现在改成只有“mousedown 和 click 都明确起始于遮罩本身”时才关闭，避免文本选择把弹窗误关掉
- **Copilot 默认模型改为 `gpt-4.1`，设置页去掉“自动”模型项** — 这轮把 Copilot 的默认模型从空值/自动改成 `gpt-4.1`，让开箱即用直接落到免费可用模型；同时设置页的 Copilot 模型选择不再显示“自动”项，功能节点里若不单独覆盖，也会明确显示“跟随全局（gpt-4.1）”而不是再给一个含混的“自动”

## [2.1.0-alpha.134] — 2026-04-20

- **修复普通文件节点 footer 被内容区挤出边界的回归** — 根因不在文件节点自身，而在 `DataNode` 共享骨架：之前为 blueprint placeholder 自动测高加入的中间 wrapper 实际包住了所有节点，但只有占位节点走了特殊高度逻辑，普通文件节点这层 wrapper 没有 `flex: 1 / minHeight: 0` 约束，导致 header/body 会把底部 AI 提示 / 文本统计区挤到节点可视边界外；现在已把非占位节点的共享 wrapper 也补齐标准 flex 约束，让 footer 重新固定在节点内部

## [2.1.0-alpha.133] — 2026-04-20

- **输入占位外壳继续向输出占位收口** — 这轮直接改占位骨架本身：输入占位的箭头改为和整体左→右工作流一致的右向表达；同时去掉输入占位主体里那块仍显得过重的独立内卡，让主体信息区更接近输出占位的简洁状态表达，不再让输入占位看起来像另一种节点

## [2.1.0-alpha.132] — 2026-04-20

- **输入占位继续往输出占位的交互骨架收口** — 这轮不再只调卡片外观，而是把输入占位的主体状态表达改得更接近输出占位：主卡片现在直接显示当前输入状态，而不再像独立说明面板；同时补回“定位输入节点”按钮，让输入占位也具备更接近输出占位那种围绕槽位做定位/交互的入口

## [2.1.0-alpha.131] — 2026-04-20

- **输入占位样式继续向输出占位收口** — 输入占位不再维持那种独立的大说明卡 + 压缩 footer 的特殊版式，而是改回和输出占位更接近的标准占位结构：主体信息收成更轻的状态卡，底部摘要区恢复显示，整体更像稳定的语义锚点，而不是单独的信息面板

## [2.1.0-alpha.130] — 2026-04-20

- **继续修蓝图重开首帧跳变，这次直接统一容器 bounds 口径** — 之前虽然运行态的 blueprint 容器重算已经不再把外部 `blueprint_bound_*` 输出节点算进蓝图内部，但加载期 `normalizeBlueprintContainerBounds` 仍旧会把这些外部 bound 节点包进外框里，导致打开时蓝图先把真实输出包住，随后再在后续重算阶段缩回去，形成“输出没动、蓝图壳子跳了一下”的视觉跳变；现在初始化和运行态都统一只按真正的 `blueprint_instance_id` 内部成员计算容器 bounds

## [2.1.0-alpha.129] — 2026-04-20

- **蓝图输出首帧跳变继续往初始化链路前移** — 加载旧 `.rsws` 时，如果蓝图容器自己还没存下 `output_slot_defs`，现在会直接从实例内的 terminal function 结构合成隐式 output slot，再让后续 runtime-output 绑定恢复、placeholder 恢复和支撑边归一化都在首帧前吃到同一份槽位语义；这样像 `research.rsws` 这类旧测试画布，不必等 definitions 异步回填后再把真实输出从蓝图内搬到外侧
- **输入占位视觉开始向输出占位收口** — 输入占位的说明区改成更接近输出占位的紧凑状态卡，减少“说明大卡片”的堆料感；同时蓝图占位节点的输入/输出通道 hover 光环也统一加强，尽量把输入占位的通道观感拉齐到输出占位

## [2.1.0-alpha.128] — 2026-04-20

- **蓝图输出初始加载跳变继续收口** — 加载期的 `normalizeBlueprintOutputSupportEdges` 现在除了补输出支撑边，还会直接把已绑定的真实输出节点预先摆到 output placeholder 外侧；这样即使旧 `.rsws` 里还保存着蓝图内位置，首帧归一化也会先把它矫正到外侧，继续减少“刚打开先在蓝图内，1-2 秒后再跳出去”的视觉跳变

## [2.1.0-alpha.127] — 2026-04-20

- **蓝图输出拓扑继续改成“功能节点 → 输出占位 → 真实输出”** — 真实输出节点不再同时直接挂在功能节点和输出占位之间；运行时与宿主写盘现在都会优先把 `ai_generated` 连到 output placeholder，再由 output placeholder 连到真实输出节点，进一步对齐输入占位只做传递/锚点的设计语义
- **修复重开画布时输出先出现在蓝图内、随后再跳到外侧的闪动** — 宿主在写盘时就会按 output placeholder 外侧的位置落点真实输出，不再先按 placeholder 位置写进 `.rsws`，从而减少重开画布后 1-2 秒内从蓝图内跳到蓝图外的迟到校正

## [2.1.0-alpha.126] — 2026-04-20

- **蓝图真实输出开始固定落到蓝图外侧** — output placeholder 继续保留在蓝图内部作为稳定语义锚点，但运行产出的真实输出节点不再贴在 function 和 placeholder 中间，而是默认放到 output placeholder 右侧外部，并继续通过绑定边连回 placeholder
- **蓝图外框 bounds / 整实例拖拽不再把外部真实输出一起包进去** — blueprint 容器重算边界和标题栏拖拽现在只跟随真正的内部成员（`blueprint_instance_id`），不再把 `blueprint_bound_instance_id` 的外部输出节点也算成蓝图内部成员；这样真实输出会保持在蓝图外，而不是又被外框吞进去

## [2.1.0-alpha.125] — 2026-04-20

- **修复 alpha.124 打开画布即崩的回归** — `research.rsws` 路径排查过程中引入的隐式 output slot 恢复逻辑，误把 `normalizedDefinition` 变量带进了未声明作用域，导致 definitions 迁移阶段直接抛 `ReferenceError`；现在已改回在正确作用域里使用标准 `definition` / 局部归一化对象，画布可重新正常打开

## [2.1.0-alpha.124] — 2026-04-20

- **直接定位到 `research.rsws` 的根因：部分蓝图定义文件本身没有 output slot** — 这不是单纯“重开后裁边”那么简单；像 `简单测试.blueprint.json`、`画图1.blueprint.json` 这类测试蓝图，定义里只有 input / pipeline，没有任何 `output_slots`，所以运行出来的最终结果节点天然拿不到 blueprint output 身份，重开后当然也没有稳定槽位可挂
- **开始为“终点 function 没显式输出节点”的蓝图自动合成 output slot** — 无论是新建蓝图、基于实例回灌蓝图，还是旧蓝图 definitions 在加载期恢复时，只要检测到“某个终点 function 没有下游 pipeline、也没有显式 data/output 节点承载结果”，系统都会自动为它补一个隐式 output slot，并补对应的 function → output_slot 语义边；这样这类 blueprint 的最终结果终于有正式输出槽位可绑定和恢复

## [2.1.0-alpha.123] — 2026-04-20

- **继续补蓝图输出 placeholder 向外连线被旧白名单裁掉的问题** — 之前 definitions 迁移期的 runtime-support 判定只对白名单里的 input placeholder / bound node 放行，导致用户如果把蓝图输出槽位 placeholder 直接连到蓝图外，下次重开画布时这类 `data_flow` 外连线仍可能被旧裁边逻辑剪掉；现在 output placeholder 的合法外连线也会被视为 blueprint 正式支撑边保留下来

## [2.1.0-alpha.122] — 2026-04-20

- **继续拆蓝图重开后的旧裁边机制** — definitions 迁移阶段之前只把“碰到 blueprint 内部节点”的边当成待审核对象，某些已经写盘成功的 bound output 支撑边可能在后续迁移/清洗里又被当残留边剪掉；现在迁移阶段会把 blueprint bound output 也纳入正式结构审查范围，不再只按内部 placeholder / function 节点白名单裁边
- **加载期新增蓝图输出支撑边自修复** — 新增 `normalizeBlueprintOutputSupportEdges`：只要画布里还能识别出合法的 bound output、output placeholder 和来源 function，就会自动补回 `function -> bound output` 的 `ai_generated` 边，以及 `bound output -> placeholder` 的 `data_flow` 绑定边，继续收口“刚打开还有、归一化一跑又掉线”的旧链路

## [2.1.0-alpha.121] — 2026-04-20

- **蓝图最终输出重开后继续掉线，改成“稳定槽位骨架 + 显式绑定边”结构** — 单输出槽位运行时不再把 output placeholder 直接替换掉，而是始终保留输出占位骨架，只更新当前 bound output，并继续把旧当前输出上的外部连线重绑到新结果；这样重开画布后不再依赖临时重建“谁才是槽位本体”
- **宿主写盘开始把 output bound node → placeholder 的绑定边一起持久化** — Extension Host 把蓝图输出写回 `.rsws` 时，不再只落 `ai_generated` 边和 `blueprint_bound_*` 元数据，而是同步把输出节点连回真实 output placeholder；旧/新画布都能更稳定地直接恢复最终输出连线

## [2.1.0-alpha.120] — 2026-04-20

- **蓝图输出切换时不再把跨蓝图外连线一起剪掉** — 单输出槽位归一与运行态替换当前输出时，现在会把旧当前输出节点上挂着的外部连线、`ai_generated` 边和 placeholder 绑定边统一重绑到新当前输出，而不是在删除旧节点时把这些边顺手一起删掉；继续收口“蓝图向外连线运行几轮或重开后全部掉线”的结构问题
- **direct pipeline 的 blueprint runtime 中间结果开始按隐藏内部节点处理** — 对于 blueprint 内部 function 之间只用 `pipeline_flow` 直连、没有显式中间结果节点的链路，运行时生成的中间结果现在会以 `blueprint_runtime_hidden` 内部节点形式落盘和加载恢复，不再像普通输出一样弹到蓝图外；旧文件打开时也会尝试把这类 raw `ai_generated` 节点自动回补成隐藏内部 runtime 结果

## [2.1.0-alpha.119] — 2026-04-20

- **蓝图最终输出重开后掉线继续做结构修复** — 这轮不再只靠 webview 在 `aiDone` 后事后补状态；Extension Host 在把输出节点写回 `.rsws` 时，也会直接继承蓝图输出槽位的 `blueprint_bound_*` 元数据与基础蓝图身份，减少“磁盘先落一版裸 ai_generated 输出、之后 definitions 迁移把它当普通节点剪掉”的竞态
- **旧画布里的 raw blueprint 输出开始加载期自动回补绑定身份** — 新增一轮加载期修复：如果发现 blueprint 内部 function 的 `ai_generated` 输出落到了画布外普通节点上，但同函数能唯一反推出目标 output slot，就会把该节点回补成正式 `bound output`；从而继续降低“刚打开时线还在，definitions/归一化一跑就掉线”的旧文件残留问题

## [2.1.0-alpha.118] — 2026-04-20

- **修复蓝图最终输出在重开画布后误掉线的问题** — 加载期归一化现在遇到“蓝图槽位定义尚未水合”的实例时，会先保留已有的 `blueprint_bound_*` 输出/输入绑定元数据，不再抢先把当前输出节点当成无效残留清掉；同时“当前输出去重”也会等真实槽位 multiplicity 到位后再执行，减少重启后最终输出看起来从输出槽位脱离、或多输出实例被提前误降级的情况

## [2.1.0-alpha.117] — 2026-04-20

- **修复顶栏“画板”按钮二次点击无法关闭的问题** — 之前画板下拉打开后，再点同一个按钮时，外部点击监听会先把面板关掉，紧接着按钮点击又按新状态重新打开，造成“怎么点都关不上”；现在按钮被排除在外部点击关闭判定之外，二次点击会正常关闭面板

## [2.1.0-alpha.116] — 2026-04-20

- **输入/输出占位开始按内容自动扩高** — 占位节点在内容变化后会自动测量自身需要的高度，并同步更新节点尺寸与蓝图外框 bounds，不再因为信息稍多就直接超出边界、被裁掉或消失
- **占位边界适应性继续收口** — 这轮不再只靠固定尺寸和隐藏滚动条硬压布局，而是让占位边界跟随实际说明内容自适应，减少“去掉滚动条后文字又被吃掉”的回归
- **蓝图占位与外框联动更新** — 占位节点自动扩高后，会继续联动蓝图实例边界重算，避免占位节点本体虽然变高了，但蓝图外框仍停在旧尺寸

## [2.1.0-alpha.115] — 2026-04-20

- **输入占位节点去掉内部滚动条** — 输入占位现在改成紧凑静态布局，不再把内容区和底部摘要拆成两层滚动/重复信息区，避免节点本体里出现难看的滚动滑块
- **占位信息继续做减法** — 输入占位底部重复摘要区已移除，主要信息收敛为：顶部标题区 + 中部输入要求卡片；“当前绑定/等待连接”也合并回主卡片，不再上下重复说同一件事
- **占位提示改成更像“页面内说明卡”而不是“堆料面板”** — 说明文案做了压缩，绑定说明限制行数，减少越界、发黑和信息挤压感

## [2.1.0-alpha.114] — 2026-04-20

- **连接失败提示不再误混成“网络错误”** — 之前因为错误分类把“连接失败”里的“连接”字样误判进了 network 分支，导致槽位类型不匹配时也会套用网络错误标题和 Ollama 提示；现在连接校验失败会独立归类成“连接失败”，不再和网络提示混用
- **蓝图输入占位样式开始做减法收口** — 输入占位节点不再同时堆叠过多重复信息；“接受类型”保留为顶部主卡片，底部只保留当前槽位与绑定摘要，减少信息重叠、越界和视觉发黑发脏的问题
- **占位节点不再展示无意义的 AI 可读提示** — 蓝图输入/输出占位本身不是实际文件，不再继续渲染 AI 可读字数/全文长度之类的信息，降低占位节点底部噪音

## [2.1.0-alpha.113] — 2026-04-20

- **蓝图输入占位的“接受类型”提示改成前置强提醒** — 输入占位节点现在会在内容区顶部显式渲染“连接前请先看输入要求”卡片，把接受类型改成更醒目的类型胶囊，并同时显示必填/可选、单绑定/多绑定，避免用户还要到节点底部慢慢找一行浅提示
- **连错类型不再只有“连不上”的黑盒感知** — 连接拖拽结束但校验失败时，现在会按真实原因弹出明确提示，例如“这个槽位只接受图像，你拖入的是笔记文本”“当前槽位只允许 1 个输入”“请拖到蓝图输入槽位圆点上”等，减少用户把失败误判成系统不稳定
